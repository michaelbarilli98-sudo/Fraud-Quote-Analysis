# Fraud Quote Analysis

A personal Python project that analyses synthetic insurance quote data to identify behavioural patterns and potential fraud indicators.

I built this project to explore how quote activity can be analysed across customer sessions and how changes to information may be used to identify cases that require further review.

The tool uses Pandas to clean and organise quote data, compare activity within quote sessions and apply simple rule-based checks.

## Features

- Tracks changes to customer information across quote sessions
- Identifies changes to occupation and annual mileage
- Detects premium reductions following quote changes
- Reviews repeated quote activity within short periods
- Applies rule-based fraud indicators
- Assigns a risk score and review priority
- Exports the analysed results to a CSV file

## Technologies

- Python
- Pandas

## How it works

The tool reads quote data from a CSV or Excel file and standardises the column names and data types.

Quote activity is grouped by customer and session so that changes made during the quote journey can be compared.

The analysis then applies a number of rule-based checks to identify activity that may warrant further investigation.

The results are exported to a new CSV file containing the original quote information alongside the identified indicators, risk score and review priority.

## Example output

When run against the included synthetic sample data, the tool identifies changes made during quote sessions and assigns a review priority.

For example:

- Customer C003 changed occupation during the same quote session
- The quote premium reduced following the change
- Risk score: 5
- Review priority: Medium

Records without identified indicators may receive a Low review priority.

The output is intended to support manual investigation rather than determine whether fraud has occurred.

## Project structure

```text
Fraud-Quote-Analysis/
│
├── README.md
├── fraud_tool.py
├── requirements.txt
├── sample_quotes.csv
│
└── results/
    └── quote_level_fraud_analysis.csv
```

`fraud_tool.py` contains the main analysis logic.

`sample_quotes.csv` contains synthetic quote activity used to demonstrate the tool.

The `results` folder contains the analysed output generated by the script.

## Data

This project uses entirely synthetic data.

It does not contain employer data, customer information, internal fraud rules or proprietary systems.

The sample data was created solely to demonstrate the functionality of the project.

## Limitations

The tool does not determine whether fraud has occurred.

The rules and risk scores are simple analytical indicators designed to highlight records that may warrant further investigation and human review.

The project is intended as a personal learning and portfolio project rather than a production fraud detection system.

## Running the project

Install the required packages:

```bash
pip install -r requirements.txt
```

Run the analysis against the included sample dataset:

```bash
python fraud_tool.py --input sample_quotes.csv --output-dir results
```

The analysed output will be saved to:

```text
results/quote_level_fraud_analysis.csv
```
